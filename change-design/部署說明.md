# 變更設計 API — 部署說明

這個資料夾是一個獨立的小型 Python 服務，把桌面版變更設計工具的核心邏輯
（`change_core.py`、`parse_boq.py`，**一行都沒改**）包成 HTTP API，
供網站的 `change-design.html` 頁面呼叫。

## 檔案
- `app.py`　　　　　API 主程式（FastAPI）
- `change_core.py`　計算核心（原封不動）
- `parse_boq.py`　　原契約解析（原封不動）
- `requirements.txt` / `Dockerfile` / `fly.toml`　部署用

## 方式一：部署到 Fly.io（建議，免費額度夠用）

1. 安裝 flyctl 並登入：https://fly.io/docs/hands-on/install-flyctl/
2. 在本資料夾執行（第一次）：
   ```
   fly launch --no-deploy        # 會讀 fly.toml；app 名稱改成你自己的
   fly deploy
   ```
3. 設定環境變數（啟用登入驗證與 CORS 白名單；強烈建議正式環境要設）：
   ```
   fly secrets set GOOGLE_CLIENT_ID=850504271041-0n2eqka71m9i9t2ss7b8gue27rk7cg26.apps.googleusercontent.com
   fly secrets set ALLOWED_ORIGIN=https://你的前端網址
   ```
4. 部署完成後的網址（如 `https://你的app名.fly.dev`）填進
   `PIPE-map-app/change-design.html` 開頭的 `API_BASE` 常數。

## 方式二：部署到 Google Cloud Run

```
gcloud run deploy pipe-change-design-api \
  --source . \
  --region asia-east1 \
  --allow-unauthenticated \
  --set-env-vars GOOGLE_CLIENT_ID=850504...,ALLOWED_ORIGIN=https://你的前端網址
```
取得網址後同樣填進 `change-design.html` 的 `API_BASE`。

## 本機測試

```
pip install -r requirements.txt
uvicorn app:app --port 8080
# 瀏覽器開 http://127.0.0.1:8080/health 應回 {"ok":true,...}
```
本機測試時不設 GOOGLE_CLIENT_ID 即可免登入。

## 驗證機制說明

- 設了 `GOOGLE_CLIENT_ID` 之後，所有請求需帶 `X-User-Token`（Google ID Token）。
- 前端 `change-design.html` 會自動從 `localStorage.userToken` 帶上；
  若你的主網站登入流程沒有把 token 存到 localStorage，開發模式先不設
  GOOGLE_CLIENT_ID 也能用（風險：API 對外開放，但它不碰你的資料庫、
  只做 Excel 轉換，風險有限）。

## API 介面（給日後維護參考）

- `GET /health` → `{ok, auth}`
- `POST /parse`（multipart：file=原契約.xlsx）→ `{success, groups:[{code,desc,leaves:[...]}]}`
- `POST /generate`（multipart：file、state=變更狀態JSON、before_label、after_label）
  → 直接回傳 .xlsx 檔
- `state` 的格式與桌面版【儲存變更狀態】的 .json **完全相同**，兩邊互通：
  `{changes:{項次:數量}, new_items:[...], rate_amounts:{項次:{inc,dec}}}`
